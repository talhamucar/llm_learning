## Tasks

### 1. Create `.gitignore`
Standard Python + Jupyter `.gitignore` covering:
- `.env` (secrets)
- `__pycache__/`, `*.pyc`
- `.venv/` (uv virtual environment)
- Jupyter checkpoints: `.ipynb_checkpoints/`
- `*.egg-info/`, `dist/`, `build/`
- Common OS files: `.DS_Store`, `Thumbs.db`

### 2. Create `.env.example`
Template file documenting required API keys (safe to commit):
```env
OPENAI_API_KEY=your_openai_key
ANTHROPIC_API_KEY=your_anthropic_key
TAVILY_API_KEY=your_search_api_key
```

### 3. Create `pyproject.toml` (uv)
A `uv`-compatible `pyproject.toml` with:
- Project metadata (name, version, Python version)
- Core dependencies: `jupyter`, `python-dotenv`, `openai`, `anthropic`, `google-generativeai`, `gradio`, `chromadb`, `faiss-cpu`, `sentence-transformers`, `transformers`, `peft`, `datasets`
- Optional groups (e.g., `[dev]` with `ipykernel`)

### 4. Create Module Folders with Notebooks
Each module folder gets one minimal `.ipynb` file per sub-topic. Each notebook will have:
- A markdown cell with the topic title and brief description
- An empty code cell ready for use

**Module 1 — `01_local_llm_api_tokenization/`**
- `local_inference.ipynb` — Running Ollama/LM Studio locally
- `cloud_apis.ipynb` — OpenAI, Anthropic, Gemini API integration
- `tokenization.ipynb` — BPE, SentencePiece, token limits

**Module 2 — `02_gradio/`**
- `ui_prototyping.ipynb` — Building quick Gradio UIs
- `chat_interfaces.ipynb` — Chat history and streaming
- `interactive_demos.ipynb` — Document uploads, parameter tuning

**Module 3 — `03_rag/`**
- `vector_databases.ipynb` — ChromaDB, Pinecone, FAISS
- `embeddings.ipynb` — OpenAI vs HuggingFace embeddings
- `knowledge_retrieval.ipynb` — Semantic search, re-ranking

**Module 4 — `04_fine_tuning/`**
- `dataset_prep.ipynb` — JSONL curation and formatting
- `peft_lora.ipynb` — PEFT/LoRA training
- `evaluation.ipynb` — Base vs fine-tuned comparison

**Module 5 — `05_agents/`**
- `function_calling.ipynb` — Tool use and external APIs
- `autonomous_loops.ipynb` — ReAct agents
- `multi_agent_systems.ipynb` — Multi-agent coordination

### 5. Update `README.md` Getting Started Section
Replace the `pip/venv` setup instructions with `uv` equivalents:
```bash
# Install uv (if not already installed)
pip install uv

# Create virtual environment and install dependencies
uv sync

# Activate environment (optional, uv run handles this)
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
```
Also update step 2 to reference `pyproject.toml` instead of `requirements.txt`.

---

## Notes
- All `.ipynb` files will be minimal valid notebooks (not empty/corrupt JSON).
- The `.env` file itself is NOT created (it's in `.gitignore`); only `.env.example`.
- `uv` will manage the virtual environment in `.venv/` (already covered by `.gitignore`).
